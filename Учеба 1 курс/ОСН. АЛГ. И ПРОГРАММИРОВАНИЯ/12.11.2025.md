# **Функции**
Вспомогательный алгоритм - это алгоритм предназначенный для использования других алгоритмов

применение:
- выполнение одинаковых действий в разных местах программы
- разбивка задачи на под задачи для лучшего понимания

>Алгоритм из которого происходит обращение, к вспомогательному алгоритму называют основным алгоритмом или вызывающим алгоритмом. Если в самом вспомогательном алгоритме используются другие вспомогательные алгоритмы то по отношению к ним данный вспомогательный алгоритм является основным.


## Два подхода к проектированию
1. низ ходящее проектирование ( сверху вниз ) - подход при котором разработка начинается с определения целей решения проблемы с последующей детализацией ( от общего к частному )
2. восходящее проектирование ( снизу вверх ) - подход при котором разработка идет от частного к общему

В большинстве случаев применяют оба подхода:
- сначала выполняют проектирование сверху вниз чтобы понять что от каждого компонента требуется и как он используется
- для разработки общих хорошо отлаженных 
- после чего возвращаются к не исходящему подходу для объеденения подходных блоков в одно целое

## Исходящая проектирование 
1. выделить главную задачу, поделить на независимые подзадачи (блоки) 
2. определить связи между блоками продумать средства взаимодействия каждого блока представить их как выходные и входные данные
3. разработать структуру каждого блока
4. записать все блоки в форме вспомогательных алгоритмов

## Восходящее проектирование
### Порядок разработки:
1. выделить в анализируемых алгоритмах одинаковые или похожие 
2. найти в них общее и отличия
3. дать отличиям обозначения и использовать в качестве выходных и входных данных для проектируемого вспомогательного алгоритма
4. общее записать в виде вспомогательного алгоритма 

## Функции
на языке Си вспомогательные алгоритмы программируются с помощью функций 
	Функция - это самостоятельная именованная единицы программы реализующая некоторый законченный вспомогательный алгоритм 

Функция может вызываться из любой части программы столько раз, сколько нужно

Главное функции программы - main не может быть повторно вызвана в этой же программе

Если тип результата отличается от void то в теле функции обязательна должна присутствовать инструкция return возвращающая обратно.

==**функция может быть определена в любом месте программы но не в теле другой функции**== 
## вызов функции
### Особенности
- вызов функции это выражение которое может являться операндом в более сложном выражении.
- независимо от того возвращает ли функция значение вызов функции может быть самостоятельный инструкцией-выражением 
- при вызове функции управление передается из вызывающей функции на первую инструкцию вызываемой функции
- по команде return или при достижении конца тела функции управление передаётся обратно в точку вызова

## В языках программирование существуют два способа передачи параметров
1. по значению - функция работает с собственным экземпляром значений аргументов любые изменения значений параметров внутри функции никак не отобразятся на соответствующих значений аргументов
2. по ссылке - вызывающая и вызываемая функция работают с одним и тем же блоком памяти
==В языке Си передача параметров в функцию осуществляется только по значениям.== 

### Особенности
1. формальные параметры - локальные переменные функции которые создаются при её вызове и инициализируются значениями фактических параметров 
2. все значения передаваемые в функцию являются для неё входными 
3. аргументами функции могут являться любые выражения тип значений которых совместим по присваиванию с типом соответствующего формального параметра
