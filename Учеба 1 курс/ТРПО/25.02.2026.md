# Современные принципы и методы разработки программных приложений 

## Сбор информации
Сбор данных - первый этап в алгоритме принятий решений

## Виды информации
**Первичная инфа** - инфо, собранная впервые для какой-либо конкретной цели
**Вторичная инфа** - инфа, которая уже существует, будучи собранной ранее для других целей

## Методы сбора данных
- Опросы и анкеты - Позволяет получить информацию непосредственно от респодентов
- Наблюдение - фиксирует поведение объектов иследования в естественных условиях
- Эксперименты - позволяют исследователю контролировать условия и переменные, чтобы изучить причинно-следственные связи
- Веб-скрапинг - автоматизированный метод сбора данных с веб-сайтов
- Анализ документов (вторичные данные) - включает в себя изучение существующих текстом, ответов, статей, и других письменных источников.

## На что влият выбор
- Определяет качество и точность собранных данных
- Убеждает в том, что данные являются релевантными, валидными и надежными.
- Помогает уменьшить смещение и повысить репрезентативность выборки
- Необходимость для принятия обоснованных решений и точных выводов
- Способствует достижению целей исследования путем предоставления точных данных
- Поддерживает обоснованность и надежность результатов исследования.

## Каскадная
![[image-157.png]] 

- Недостатки модели
	Жесткое ограничение последовательности действий по разработке

	Игнорирование меняющихся нужд пользователей, факторов

- Целесообразность применения: комплексные системы, для которых долгий цикл сопровождения более важен, чем затраты на разработку.

## Инкрементная
![[image-158.png]]

Недостатки
- Требование фиксированы на протяжении всего процесса разработки
- Целесообразность применения
	Необходима быстрая реализация возможностей системы

	Существует декомпозиция системы на составляющие части, реализуемые как самостоятельные промежуточные или готовые продукты

	Возможно увеличение финансирования на разработку отдельных частей продукта

## Спиральная 
![[image-159.png]]

**Недостатки**
- Проблема выбора момента начала новой итерации
**Целесообразность**
- Комплексные дорогостоящие проекты, для которых критически важна минимизация риска

**Преимущества**
- Контроль риска раннее тестирование наиболее сложных частей ПО
- Выбор распределения между работами (анализ требований, проектирования, создание прототипов, тестирование)
- Выбор уровня детализации
- Выбор модели процесса разработки (каскадная модель, прототипирование) на каждой итерации

## Эволюционная
![[image-160.png]]

- Создание множества прототипов продукта для определения и уточнения требований пользователей
- Интенсивное использование средств автоматизации

**Недостатки**
- Дополнительные затраты на разработку прототипов
- Недостаточный анализ системы (=>неоптимальная архитектура)
- Риск интерпретации заказчиком прототипов как финального продукта
**Целесообразность применения** 
Проекты, для которых важен пользовательский интерфейс

## Гибкая
![[image-161.png]] **Методы**
- Непрерывная интеграция
- Проблемно-ориентированное; проектирование
- Парное программирование
- Разработка через тестирование
- Автоматизированное модульное тестирование
- Шаблоны проектирования
- Рефакторинг кода

**Недостатки**
- Неоптимальная архитектура системы вследствие отсутствия требований, фиксированных на протяжении всего процесса разработки
- Риск снижения качества продукта из-за множества изменений, вносимых без достаточного тестирования и проверки на соответствие общей архитектуре системы
**Целесообразность применения**
- Программные проекты с необходимостью частых выпуском
- ПО с быстро меняющимися требованиями

## RAD
![[image-162.png]]

- Инструментарий должен быть нацелен на минимизацию времени разработки
- Создание прототипа для уточнения требований заказчика
- Цикличность разработки каждая новая версия продукта основывается на оценке результата работы предыдущей версии заказчиком
- Минимизация времени разработки версии, за счет переноса уже готовых модулей и добавления функциональности в новую версию
- Команда разработчиков должна тесно сотрудничать, каждый участник должен быть готов выполнять несколько обязанностей
- Управление проектом должно минимизировать длительность цикла разработки

RAD - технология не является универсальной, её целесообразно применять лишь если проект отвечает всем или некоторым из условий
- Сжатые сроки
- Нечетко определенные или изменяющиеся по ходу разработки требования
- Органиченный бюджет при готовности участия заказчика в разработке
- Небольшие объемы либо возможность разбиения проекта на функциональные компоненты
- Графический интерфейс пользоватлея
- Низкая вычислительная сложность

# Основные принципы моделирования
- Декомпозиция (разделяй и властвуй)
- Иерархического упорядочения
- Концептуальной общности
- Абстрагирования
- Формализации
- Унификации
- Логической независимости
- Многомодельности
- Непротиворечивости (согласованности)
- Информационной закрытости (инкапсуляция)
- Полиморфизма
