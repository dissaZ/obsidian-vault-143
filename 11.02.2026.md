## Объектно-ориентированное проектирование 
 - решение задачи проектирования программный системы с использованием объектов и взаимодействий между ними

Объект - сильная связь между структурами данных и методами, обрабатывающими эти данные

Составляющие объекта
- Идентификатор
- Свойства
- Методы

### **Концепции** **ООП**
- Объекты
- Инкапсуляция - информационная закрытость (public or private)
- Наследование
- Дополнение функциональности объектов
- Интерфейсы
- Разделение описание и имплементации
- Полиморфизм

## **Входные данные для ООП**
- Концептуальная модель (диаграмма классов UML с основными понятиями предметной области, независимая от реализации)
- Варианты применения
- Диаграммы последовательности для вариантов применения
- Реляционная модель данных  (может разрабатываться параллельно с объектами)

## Процесс ООП
- Разграничение объектов составление диаграммы классов (сущность => объекты)
- Конкретизация диаграмм последовательности
- Определение контекста (библиотек, КПИ ...)
- Подбор и реализация шаблонов проектирования
- Определение взаимодействия объектов с источниками данных ( базы данных, удаленные объекты)

## Отличия структурного и объектно-ориентированного подхода
- В принципах декомпозиции и структурной организации элементов ( компонентов, модулей) системы
- Объединение в объекте как атрибутивных данных ( характеристик, свойств), так и поведения (функций, методов)
- Структурной организации внутри модулей системы

## Основы объектно-ориентированного подхода
Методологии, поддерживающие подход:
- Унифицированный процесс ( Unified process, UP)
- Экстремальное программирование (eXtreme Programming, XP)
- Гибкое моделирование (Agile Modeling AM)

Базовое средство документирования - Унифицированный язык моделирования

# В ООП есть 3 основных принципа

- Наследование
- Инкапсуляция
- Полиморфизм

## Базовые составляющие ООП

Унифицированный процесс - процесс разработки программного ПО который обеспечивает упорядоченный подход к распределению задач и обязанностей в организации-разработчике

Унифицированный язык моделирования - язык (система обозначений) для определения, визуализации и конструирования моделей системы в виде диаграмм и документов на основе объектно-ориентированного подхода

Шаблоны проектирования - именованные пары "проблема/решение", содержащие готовые обобщенные решения типичных проблем

## Преимущества ООП
- Описание системы в виде объектов больше соответствует содержательному смыслу предметной области
- Сущности реального мира, как правило, обладают поведением, что в ООП отражается с помощью определения методов класса
- Объединение атрибутов и методов в объекте (классе), а также инкапсуляция позволяют добиться большей внутренней и меньшей внешней связности между компонентами системы.

Это облегчает решение проблем
- Адаптации системы к изменению существующих или появлению новых требований
- Сопровождения системы на разных стадиях жизненного цикла
- Повторного использования компонентов

ООП подход позволяет легче организовать параллельные вычисления

# UML
Унифицированный процесс - методология, содержащая детальное описание работ по созданию и внедрению ПО.

## Структура UML 
Она отвечает на вопросы **когда, как, кто, что и с помощью чего реализуется проект**,
А именно содержит описание:
- Технологических процессов (когда)
- Видов деятельности (как)
- Исполнителей (кто)
- Артефактов (что)
- Используемых утилит (с помощью чего)

## **Немного истории**

Унифицированный язык моделирования в настоящий момент является стандартом де-факто при описании (документирования) результатов пректирования и разработки ООП
